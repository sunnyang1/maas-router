Introduction

Welcome to the AI API — a powerful REST interface for integrating state-of-the-art language models into your applications. Build chat-based assistants, generate content, analyze text, and more with a simple, consistent API.

● https://api.ai-service.com/v1
ℹ
Quick Start
All API requests must be made over HTTPS. Requests over plain HTTP will be rejected. The API accepts JSON request bodies and returns JSON-encoded responses with standard HTTP status codes. Authentication is handled via Bearer tokens in the Authorization header.

Authentication

The AI API uses API keys to authenticate requests. You can generate and manage your API keys from the dashboard. All API requests must include your key in the Authorization header.

Bearer Token

Include your API key as a Bearer token in every request:

Header Authorization: Bearer <your-api-key>

Replace <your-api-key> with your actual API key. Keep your key secret — do not expose it in client-side code or public repositories.

⚠
Security Best Practice
Store API keys in environment variables or a secure secrets manager. Never commit them to version control. If a key is compromised, revoke it immediately from the dashboard and generate a new one.

Chat Completions

Create a model response for a given chat conversation. This is the primary endpoint for interacting with language models — send a list of messages and receive a generated completion in return.

POST /v1/chat/completions

Generates a model response for the given conversation. The model will process the message history and return a coherent completion that continues the dialogue.

Request Parameters
Parameter Type Description
model required string ID of the model to use. See the Models section for available models.
messages required array A list of messages comprising the conversation so far. Each message has a role (system, user, assistant) and content.
temperature optional number Sampling temperature between 0 and 2. Higher values produce more random outputs. Default: 1
max_tokens optional integer The maximum number of tokens to generate in the completion. Default: model maximum
top_p optional number Nucleus sampling parameter. Only tokens with cumulative probability above this threshold are considered. Default: 1
stream optional boolean If true, the response is streamed as server-sent events (SSE). Default: false
stop optional string | array Up to 4 sequences where the API will stop generating further tokens.
Response

Returns a 200 OK with a JSON object containing the generated completion and usage metadata.

Models

List and describe the available language models. Use these endpoints to discover which models are available for chat completions and their capabilities.

GET /v1/models

Lists all currently available models, and provides basic information about each one such as the owner and availability.

Response

Returns a 200 OK with a JSON object containing a data array of model objects.

GET /v1/models/{model_id}

Retrieves detailed information about a specific model, including its capabilities, token limits, and pricing tier.

Parameter Type Description
model_id required string The ID of the model to retrieve (path parameter). Example: gpt-4o

Error Handling

The API uses conventional HTTP response codes to indicate the success or failure of a request. Errors return a JSON object with a machine-readable type and a human-readable message.

Error Response Format

All error responses follow a consistent structure:

ERROR Response Structure
Field Type Description
error.type string A machine-readable error type identifier (e.g. invalid_request_error, authentication_error).
error.message string A human-readable description of the error, suitable for display to developers.
error.code string | null An optional machine-readable error code for programmatic handling (e.g. context_length_exceeded).
error.param string | null For parameter-related errors, the name of the parameter that caused the error.

HTTP Status Codes

Status Type Description
400 Bad Request Invalid request parameters, missing required fields, or malformed JSON.
401 Unauthorized Missing or invalid API key. Check the Authorization header.
403 Forbidden Your API key does not have permission to access the requested resource.
404 Not Found The requested endpoint or model does not exist.
429 Rate Limit Too many requests. Check the Retry-After header and implement exponential backoff.
500 Internal Error An unexpected server error occurred. Retry with exponential backoff.
ℹ
Retry Strategy
For 429 and 5xx errors, implement exponential backoff: start with a 1-second delay and double it after each retry, up to a maximum of 5 retries. Use jitter to avoid thundering herd problems.

Python — Error Handling 
import requests
import time

def api_call_with_retry(url, headers, 
                         max_retries=5):
    for attempt in range(max_retries):
        response = requests.post(url, 
                                 headers=headers)
        
        if response.status_code == 200:
            return response.json()
        
        if response.status_code == 429:
            retry_after = int(
                response.headers.get(
                    "Retry-After", "1"
                )
            )
            time.sleep(retry_after)
            continue
        
        if response.status_code >= 500:
            wait = 2 ** attempt
            time.sleep(wait)
            continue
        
        # Client error — don't retry
        raise Exception(
            f"Error {response.status_code}: "
            f"{response.json()}"
        )
    
    raise Exception("Max retries exceeded")
JavaScript — Error Handling 
async function apiCallWithRetry(
  url, headers, maxRetries = 5
) {
  for (let attempt = 0; attempt < maxRetries; 
       attempt++) {
    const res = await fetch(url, { headers });
    
    if (res.ok) return await res.json();
    
    if (res.status === 429) {
      const retryAfter = 
        res.headers.get("Retry-After") || 1;
      await new Promise(r => 
        setTimeout(r, retryAfter * 1000));
      continue;
    }
    
    if (res.status >= 500) {
      const wait = Math.pow(2, attempt);
      await new Promise(r => 
        setTimeout(r, wait * 1000));
      continue;
    }
    
    throw new Error(
      `Error ${res.status}: ` +
      JSON.stringify(await res.json())
    );
  }
  
  throw new Error("Max retries exceeded");
}
